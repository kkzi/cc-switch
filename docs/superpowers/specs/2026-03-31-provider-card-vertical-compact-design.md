# 供应商卡片垂直高度二次紧凑化设计

## 背景

上一轮已经对供应商列表整体做了轻度紧凑化，但用户希望进一步减少供应商卡片本身的垂直高度，同时保持当前列表结构、信息密度和交互逻辑不变。

本次调整只聚焦 `ProviderCard`，不再改动列表外层、搜索区、空状态或无结果状态。

## 目标

- 让供应商卡片的垂直高度再减少一档
- 保持当前信息完整，不删除字段
- 保持按钮集合、hover 行为、状态标签和故障转移逻辑不变
- 展开后的用量区一并再压缩一档，保证密度一致

## 非目标

- 不修改 `ProviderList` 外层列表间距
- 不修改空状态和无结果状态
- 不调整字体大小和文本层级
- 不重排横向布局

## 方案选择

### 方案 1：只缩卡片外层与展开区留白

优点：
- 改动最小
- 风险最低

缺点：
- 标题区和 URL 区高度下降有限

### 方案 2：只缩卡片主体内部节奏

优点：
- 主体高度会明显下降

缺点：
- 如果不动外层 padding，卡片边框内部仍会显得偏松

### 方案 3：卡片内垂直一档压缩

优点：
- 最符合“卡片再矮一档”的目标
- 范围只在 `ProviderCard`
- 风险可控

缺点：
- 需要额外确认可读性不会变差

### 结论

采用方案 3。

## 范围

受影响文件：

- `src/components/providers/ProviderCard.tsx`
- `tests/components/ProviderCard.test.tsx`

## 详细设计

### 1. 卡片外层高度

在 `ProviderCard.tsx` 中将卡片外层 padding 再下调一档，让卡片在默认折叠状态下明显更矮。

### 2. 主体纵向节奏

同步收缩以下竖向节奏：

- 卡片主体容器的纵向 `gap`
- 标题区外层 `space-y`
- 标题/标签行的最小高度

约束：

- 不改变标题字号
- 不改变 URL 字号
- 不删除任何标签或徽标

### 3. 展开用量区

展开后的用量区域同步收缩：

- 顶部外边距 `mt`
- 顶部内边距 `pt`

目标是避免折叠态和展开态的密度不一致。

## 测试策略

更新 `tests/components/ProviderCard.test.tsx`：

- 先锁定新的更紧凑卡片外层 padding
- 锁定新的主体纵向 `gap`
- 锁定展开用量区新的 `mt/pt`

执行：

- `pnpm exec vitest run tests/components/ProviderCard.test.tsx --exclude .worktrees/**`
- `pnpm typecheck`

## 风险

- 标题区和标签区可能显得偏挤
  - 通过只压缩一档且不改字号控制风险
- 展开后的用量区可读性可能下降
  - 只收缩留白，不改 `UsageFooter` 内容布局
